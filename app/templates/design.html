DESIGN PATTERNS

What are Design Patterns?

According to the book called “Design Patterns: Elements of Reusable Object-Oriented Software by GoF(Gang of Four)”. Design patterns are referred to as the Gang of Four patterns. They are considered the foundation for all other patterns. They are Categorized into three groups: Structural, Creational and Behavioural.

Design patterns describe the structure of code classes and how these classes should collaborate, building reusable and extensible software.

Learning these design patterns helps to communicate ideas with other developers at a more abstract level without writing unnecessarily long lines of codes. It also helps new developers understand new programming languages, libraries and frameworks quickly.

Structural Design Patterns:

These types of design patterns are focused on Classes and Objects compositions that are the relationships between these objects. Structural class patterns use inheritance to compose interfaces and Object patterns define ways to compose objects to obtain new functionality.

A decorator is a type of a structural design pattern. It instantiates the object and adds responsibility to the object. In the calculator code example, the values are the attributes of the calculator. This makes using values alone in the other part of the programs possible without having to type self repeatedly.

def __init__(self, values):
self.values = values

Creational Design Patterns:

Creational design patterns are all about class instantiation and creating objects. This pattern can be further divided into class-creation patterns and object-creational patterns, in class creation patterns the inheritance is used effectively in the instantiation process, while the object-creation patterns use delegation effectively to get the job done.

Factory method is one of the subdivisions of creational structure. It is used to create an instance of several derived classes. It helps us to avoid constant code modification whenever a new object needs to be created. For instance in our calculator example I have a factory function that performs different methods without needing to change the other.

def factory(self, operation):
""" Factory helper method """
if operation == 'add':
add_object = Add.create(*self.values)
result = add_object.add()
Calculator.add_calculation_to_history({
"result": result,
"object": add_object
})
return result
if operation == 'subtract':
subtract_object = Subtract.create(*self.values)
result = subtract_object.subtract()
Calculator.add_calculation_to_history({
"result": result,
"object": subtract_object
})
return result
if operation == 'multiply':
multiply_object = Multiply.create(*self.values)
result = multiply_object.multiply()
Calculator.add_calculation_to_history({
"result": result,
"object": multiply_object
})
return result
if operation == 'divide':
divide_object = Divide.create(*self.values)
result = divide_object.divide()
Calculator.add_calculation_to_history({
"result": result,
"object": divide_object
})
return result
return "Invalid Operation"

Behavioral Designs Patterns:

Behavioral design patterns focus on the communications and interactions between objects and their responsibilities in a code. They are the largest part of the design pattern. They basically show the relationships between functions.

For instance in our calculator code, the fixture of clear history is an example of a command which is a type of behavioral design pattern.

def test_clear_history(calculator_object):
""" Test clear history of calculator """
calculator_object.factory("add")
calculator_object.factory("multiply")
calculator_object.clear_history()
assert len(calculator_object.get_history()) == 0
